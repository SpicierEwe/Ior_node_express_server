Certificate API Documentation

Welcome to the API documentation for the Certificate Management System. Below you'll find details on how to use each endpoint.

GET /

Returns this very documentation.

curl -X GET http://localhost:5000/

POST /certificates

Creates a new certificate.

Request Body:

{
    "certificate_name": "Certificate Name",
    "issue_date": "2024-07-01",
    "expiry_date": "2025-07-01",
    "issued_by": "Issuer Name",
    "created_at": "2024-07-01 08:00:00",
    "updated_at": "2024-07-01 08:00:00"
}
curl -X POST http://localhost:5000/certificates -H "Content-Type: application/json" -d '{ "certificate_name": "Certificate Name", "issue_date": "2024-07-01", "expiry_date": "2025-07-01", "issued_by": "Issuer Name", "created_at": "2024-07-01 08:00:00", "updated_at": "2024-07-01 08:00:00" }'

GET /certificates

Fetches all certificates.

curl -X GET http://localhost:5000/certificates

GET /certificates/:id

Fetches a specific certificate by ID.

curl -X GET http://localhost:5000/certificates/3

PUT /certificates/:id

Updates an existing certificate by ID.

Request Body:

{
    "certificate_name": "Updated Certificate Name",
    "issue_date": "2024-07-01",
    "expiry_date": "2025-07-01",
    "issued_by": "Updated Issuer Name",
    "created_at": "2024-07-01 08:00:00",
    "updated_at": "2024-07-02 08:00:00"
}
curl -X PUT http://localhost:5000/certificates/3 -H "Content-Type: application/json" -d '{ "certificate_name": "Updated Certificate Name", "issue_date": "2024-07-01", "expiry_date": "2025-07-01", "issued_by": "Updated Issuer Name", "created_at": "2024-07-01 08:00:00", "updated_at": "2024-07-02 08:00:00" }'

DELETE /certificates/:id

Deletes a certificate by ID.

curl -X DELETE http://localhost:5000/certificates/3

Database Schema

The table structure for the certificate_tbl table:

CREATE TABLE certificate_tbl (
  id SERIAL PRIMARY KEY,
  certificate_name TEXT NOT NULL,
  issue_date DATE NOT NULL,
  expiry_date DATE,
  issued_by TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
