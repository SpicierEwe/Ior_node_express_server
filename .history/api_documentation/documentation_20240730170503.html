Certificate Management API Documentation

Welcome to the API documentation for the Certificate Management System. Below you'll find details on how to use each endpoint.

GET /

Returns this very documentation.

curl -X GET http://localhost:5000/

POST /certificates

Creates a new certificate.

Request Body:

{
    "name": "Certificate Name",
    "email": "example@example.com",
    "months": 6,
    "certificate_of": "Course Completion",
    "date": "2024-07-01",
    "profile": "Profile Info",
    "time": "12:00 PM",
    "c_type": "Type of Certificate",
    "content": "Certificate Content",
    "issued_by": "Issuer Name",
    "status": "Active",
    "time_stamp": "2024-07-01 08:00:00"
}
curl -X POST http://localhost:5000/certificates -H "Content-Type: application/json" -d '{ "name": "Certificate Name", "email": "example@example.com", "months": 6, "certificate_of": "Course Completion", "date": "2024-07-01", "profile": "Profile Info", "time": "12:00 PM", "c_type": "Type of Certificate", "content": "Certificate Content", "issued_by": "Issuer Name", "status": "Active", "time_stamp": "2024-07-01 08:00:00" }'

GET /certificates

Fetches all certificates.

curl -X GET http://localhost:5000/certificates

GET /certificates/:id

Fetches a specific certificate by ID.

curl -X GET http://localhost:5000/certificates/3

PUT /certificates/:id

Updates an existing certificate by ID.

Request Body:

{
    "name": "Updated Certificate Name",
    "email": "updated@example.com",
    "months": 12,
    "certificate_of": "Updated Course Completion",
    "date": "2024-07-02",
    "profile": "Updated Profile Info",
    "time": "1:00 PM",
    "c_type": "Updated Type of Certificate",
    "content": "Updated Certificate Content",
    "issued_by": "Updated Issuer Name",
    "status": "Inactive",
    "time_stamp": "2024-07-02 08:00:00"
}
curl -X PUT http://localhost:5000/certificates/3 -H "Content-Type: application/json" -d '{ "name": "Updated Certificate Name", "email": "updated@example.com", "months": 12, "certificate_of": "Updated Course Completion", "date": "2024-07-02", "profile": "Updated Profile Info", "time": "1:00 PM", "c_type": "Updated Type of Certificate", "content": "Updated Certificate Content", "issued_by": "Updated Issuer Name", "status": "Inactive", "time_stamp": "2024-07-02 08:00:00" }'

DELETE /certificates/:id

Deletes a certificate by ID.

curl -X DELETE http://localhost:5000/certificates/3

Database Schema

The table structure for the certificate_tbl table:

CREATE TABLE certificate_tbl (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255) NOT NULL,
  months INT,
  certificate_of VARCHAR(255) NOT NULL,
  date DATE NOT NULL,
  profile TEXT,
  time TIME,
  c_type VARCHAR(255),
  content TEXT,
  issued_by VARCHAR(255),
  status VARCHAR(50),
  time_stamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
