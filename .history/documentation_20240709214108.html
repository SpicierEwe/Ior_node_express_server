Events API Documentation

Welcome to the API documentation for the Events Management System. Below you'll find details on how to use each endpoint.

GET /

Returns a welcome message.

curl -X GET http://localhost:3000/

POST /events

Creates a new event.

Request Body:

{
    "parent_id": 1,
    "company_id": 2,
    "event_name": "Product Launch",
    "label_color": "green",
    "location": "Main Hall",
    "description": "Launch of the new product line.",
    "start_date_time": "2024-07-12T12:30:00.000Z",
    "end_date_time": "2024-07-12T14:30:00.000Z",
    "host": "Emily Davis",
    "status": "confirmed",
    "note": "Invite media personnel.",
    "repeat": 0,
    "send_reminder": 1,
    "remind_time": "17:30:00",
    "remind_type": "sms",
    "event_link": "http://example.com/event3",
    "added_by": 2,
    "last_updated_by": 2,
    "event_id": "EVT12347"
}
curl -X POST http://localhost:3000/events -H "Content-Type: application/json" -d '{ "parent_id": 1, "company_id": 2, "event_name": "Product Launch", "label_color": "green", "location": "Main Hall", "description": "Launch of the new product line.", "start_date_time": "2024-07-12T12:30:00.000Z", "end_date_time": "2024-07-12T14:30:00.000Z", "host": "Emily Davis", "status": "confirmed", "note": "Invite media personnel.", "repeat": 0, "send_reminder": 1, "remind_time": "17:30:00", "remind_type": "sms", "event_link": "http://example.com/event3", "added_by": 2, "last_updated_by": 2, "event_id": "EVT12347" }'

GET /events

Fetches all events.

curl -X GET http://localhost:3000/events

GET /events/:id

Fetches a specific event by ID.

curl -X GET http://localhost:3000/events/3

PUT /events/:id

Updates an existing event by ID.

Request Body:

{
    "event_name": "Updated Product Launch"
}
curl -X PUT http://localhost:3000/events/3 -H "Content-Type: application/json" -d '{ "event_name": "Updated Product Launch" }'

DELETE /events/:id

Deletes an event by ID.

curl -X DELETE http://localhost:3000/events/3
